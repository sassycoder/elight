Implementation notes

Gallery/Single image component

- The user should be prompted to upload a low res (thumbnail) and high res version of each image. (Low res is mandatory)
- The low res version is always displayed on the page. The high res version is only displayed in a modal window.

- The low res version should be written to the page as a standard <img> (see example below)
- The high res version should be written to the href attribute of an <a> tag. The CMS needs to obtain the dimensions of the image that is referenced in the href and write them to a data attribute (see example below)

<a href="image1.jpg" itemprop="contentUrl" data-size="1200x1210" class="gallery-thumb"> 
    <img src="image1_thumb.jpg" itemprop="thumbnail" alt="Image description" />
  </a>


- if the user does not upload a high res version, the low res version should be referenced in the <a> instead.
- The user should be able to add an optional caption heading and caption text (see example below)
- The user should be able to use HTML tags within this content

<figcaption itemprop="caption description">
        <span class="cap-heading”>Caption heading</span>
        <span class="caption”><strong>Caption</strong> text</span>
</figcaption>


The final HTML output should be as follows:

<div class="component gallery" itemscope itemtype="http://schema.org/ImageGallery">
  <figure itemprop="associatedMedia" itemscope itemtype="http://schema.org/ImageObject”>

   <a href=“image1.jpg" itemprop="contentUrl" data-size="1200x1210" class="gallery-thumb”> 
  <img src=“image1_thumb.jpg" itemprop="thumbnail" alt="Image description" /> 
   </a>

    <figcaption itemprop="caption description">
        <span class="cap-heading”>Caption heading</span>
        <span class="caption”><strong>Caption</strong> text</span>
   </figcaption>
  </figure>   
</div>

Other options

- The user should be able to specify whether the image/gallery is full-width. I would suggest a checkbox with a label of "Full width". If the user checks this box, the CMS would add a css class to the component e.g.

<div class="component gallery full-width" itemscope itemtype="http://schema.org/ImageGallery">
...

- The user should be able to specify a background colour for the image/gallery. The CMS would write the colour inline e.g.

<a href="image1.jpg" itemprop="contentUrl" data-size="1200x1210" class="gallery-thumb" style="background-color: #ebebeb;"> 
   <img src="image1_thumb.jpg" itemprop="thumbnail" alt="Image description"/>
</a>
